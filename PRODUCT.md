# Product Context

## Register

product

## Product Name

航运公司管理系统

## Product Purpose

本项目是一个航运业务课程设计后台管理系统，用于演示前后端分离开发、RESTful API、MySQL 数据建模和 Vue 3 管理端页面实现。系统服务于船舶、船员、运输任务、财务结算、字典、用户和角色等基础业务维护。

## Users

| 用户 | 目标 |
| --- | --- |
| 系统管理员 | 维护用户、角色、字典和基础系统数据 |
| 业务用户 | 查看船舶、船员、运输任务和财务结算 |
| 查看用户 | 查看业务数据和系统概览 |
| 课程答辩评审者 | 快速理解系统范围、业务流程和技术实现 |

## Product Tone

- 专业正式
- 清晰稳定
- 适合学校课程设计答辩
- 不做营销化表达
- 不做炫技式大屏和复杂动效

## Strategic Principles

1. 业务操作优先：列表、筛选、表单、状态流转和数据反馈必须清楚。
2. 课程设计适度完整：核心流程要闭环，非核心能力保持简化。
3. 航运主题克制表达：通过色彩、图标、字段和状态体现海运场景，不做夸张装饰。
4. 易演示：登录后能快速进入工作台，并完成船舶、船员、运输、结算的完整演示路径。
5. 明确简化边界：密码明文存储和简单 token 是课程设计取舍，不在界面中标成漏洞。

## Anti References

- 不做暗色监控大屏。
- 不做营销落地页。
- 不做过度玻璃拟态、霓虹渐变、夸张动效。
- 不做纯卡片墙后台。
- 不做需要复杂权限树才能完成演示的系统管理页面。

## Primary Workflows

1. 用户登录进入业务工作台。
2. 维护船舶和船员基础档案。
3. 创建运输任务并推进状态。
4. 为运输任务生成结算记录并登记收款。
5. 通过首页统计和列表筛选查看业务状态。
6. 管理字典、用户和角色。
