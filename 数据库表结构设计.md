# 简易航运公司管理系统数据库表结构设计

## 1. 设计说明

本文档用于定义简易航运公司管理系统的数据库表结构。数据库设计与需求文档、API 接口文档保持一致，主要支撑用户登录、船舶管理、船员管理、货物运输管理、财务结算管理、基础字典管理和首页统计等功能。

本项目是课程设计项目，数据库设计以结构清晰、便于实现、便于演示为目标，不追求生产级复杂度。

特别说明：

1. 用户密码直接以明文形式存储在用户表的 `password` 字段中。
2. 登录时后端直接比较用户输入密码和数据库中的明文密码。
3. 不设计密码哈希、加盐、密文版本、登录失败锁定、刷新 Token、复杂会话表等生产级安全字段。
4. 以上是基于课程设计范围和开发成本做出的明确取舍，不作为漏洞处理，也无需在本项目中优化。

## 2. 数据库基本约定

数据库类型使用 MySQL。本地当前环境为 Scoop 安装的 MySQL 9.7.0，表结构也兼容常见 MySQL 8.x 环境。

本地开发连接信息：

| 配置项 | 值 |
| --- | --- |
| 数据库名 | `shipping_management` |
| 主机 | `localhost` |
| 备用主机 | `127.0.0.1` |
| 端口 | `3306` |
| 用户名 | `gavin` |
| 密码 | `456123` |
| JDBC URL | `jdbc:mysql://localhost:3306/shipping_management?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai` |

字符集建议：

```sql
utf8mb4
```

排序规则建议：

```sql
utf8mb4_general_ci
```

命名约定：

1. 表名使用小写字母和下划线。
2. 主键统一命名为 `id`。
3. 创建时间字段统一命名为 `created_at`。
4. 更新时间字段统一命名为 `updated_at`。
5. 状态字段统一使用 `varchar` 保存枚举值，便于前后端直接对应。
6. 金额字段使用 `decimal(12,2)`。
7. 重量字段使用 `decimal(12,2)`，单位为吨。

## 3. 表关系概述

系统核心表关系如下：

1. `sys_user.role_id` 关联 `sys_role.id`。
2. `crew_member.ship_id` 关联 `ship.id`。
3. `transport_order.ship_id` 关联 `ship.id`。
4. `settlement.transport_order_id` 关联 `transport_order.id`。
5. `dictionary_item` 用于维护船舶类型、货物类型、港口、船员岗位等基础数据。

为了降低课程设计实现难度，外键可根据实际开发情况选择是否在数据库层强制创建。即使不创建数据库外键，也应在后端业务逻辑中保证关联数据有效。

## 4. 表结构详情

## 4.1 角色表 sys_role

用于保存系统角色信息，例如管理员、业务用户、查看用户。

```sql
CREATE TABLE sys_role (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '角色ID',
  role_name VARCHAR(30) NOT NULL COMMENT '角色名称',
  role_code VARCHAR(30) NOT NULL COMMENT '角色编码，如 ADMIN、BUSINESS、VIEWER',
  description VARCHAR(255) DEFAULT NULL COMMENT '角色说明',
  built_in TINYINT NOT NULL DEFAULT 0 COMMENT '是否内置角色：0否，1是',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_sys_role_code (role_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='角色表';
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 角色主键 |
| role_name | VARCHAR(30) | 是 | 角色名称 |
| role_code | VARCHAR(30) | 是 | 角色编码，系统内唯一 |
| description | VARCHAR(255) | 否 | 角色说明 |
| built_in | TINYINT | 是 | 是否系统内置角色 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

## 4.2 用户表 sys_user

用于保存系统登录用户和基础资料。

```sql
CREATE TABLE sys_user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username VARCHAR(30) NOT NULL COMMENT '用户名',
  password VARCHAR(30) NOT NULL COMMENT '明文密码，课程设计简化使用，不进行加密或哈希处理',
  real_name VARCHAR(30) NOT NULL COMMENT '真实姓名',
  phone VARCHAR(30) DEFAULT NULL COMMENT '联系电话',
  email VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
  role_id BIGINT NOT NULL COMMENT '角色ID',
  status VARCHAR(20) NOT NULL DEFAULT 'ENABLED' COMMENT '用户状态：ENABLED启用，DISABLED禁用',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_sys_user_username (username),
  KEY idx_sys_user_role_id (role_id),
  KEY idx_sys_user_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 用户主键 |
| username | VARCHAR(30) | 是 | 用户名，系统内唯一 |
| password | VARCHAR(30) | 是 | 明文密码，课程设计简化使用 |
| real_name | VARCHAR(30) | 是 | 真实姓名 |
| phone | VARCHAR(30) | 否 | 联系电话 |
| email | VARCHAR(100) | 否 | 邮箱 |
| role_id | BIGINT | 是 | 角色 ID |
| status | VARCHAR(20) | 是 | 用户状态 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

密码设计说明：

`password` 字段故意使用明文存储，后端登录接口直接进行字符串比较。这是课程设计范围内的简化实现，不需要改为 `password_hash`，也不需要增加 `salt` 字段。

## 4.3 船舶表 ship

## 4.5 运输任务表 transport_order

用于保存货物运输任务、航线安排、运输状态和客户信息。

```sql
CREATE TABLE transport_order (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '运输任务ID',
  order_no VARCHAR(40) NOT NULL COMMENT '运输任务编号',
  cargo_name VARCHAR(100) NOT NULL COMMENT '货物名称',
  cargo_type VARCHAR(50) NOT NULL COMMENT '货物类型',
  cargo_weight DECIMAL(12,2) NOT NULL COMMENT '货物重量，单位：吨',
  origin_port VARCHAR(50) NOT NULL COMMENT '起运港',
  destination_port VARCHAR(50) NOT NULL COMMENT '目的港',
  ship_id BIGINT NOT NULL COMMENT '运输船舶ID',
  customer_name VARCHAR(100) NOT NULL COMMENT '客户名称',
  customer_phone VARCHAR(30) DEFAULT NULL COMMENT '客户联系电话',
  planned_departure_time DATETIME NOT NULL COMMENT '预计出发时间',
  planned_arrival_time DATETIME NOT NULL COMMENT '预计到达时间',
  actual_departure_time DATETIME DEFAULT NULL COMMENT '实际出发时间',
  actual_arrival_time DATETIME DEFAULT NULL COMMENT '实际到达时间',
  status VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT '运输状态：PENDING待出发，IN_TRANSIT运输中，ARRIVED已到达，CANCELLED已取消',
  remark VARCHAR(255) DEFAULT NULL COMMENT '备注',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_transport_order_no (order_no),
  KEY idx_transport_order_ship_id (ship_id),
  KEY idx_transport_order_status (status),
  KEY idx_transport_order_customer_name (customer_name),
  KEY idx_transport_order_planned_departure_time (planned_departure_time),
  KEY idx_transport_order_route (origin_port, destination_port)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='运输任务表';
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 运输任务主键 |
| order_no | VARCHAR(40) | 是 | 运输任务编号，系统内唯一 |
| cargo_name | VARCHAR(100) | 是 | 货物名称 |
| cargo_type | VARCHAR(50) | 是 | 货物类型 |
| cargo_weight | DECIMAL(12,2) | 是 | 货物重量，单位为吨 |
| origin_port | VARCHAR(50) | 是 | 起运港 |
| destination_port | VARCHAR(50) | 是 | 目的港 |
| ship_id | BIGINT | 是 | 运输船舶 ID |
| customer_name | VARCHAR(100) | 是 | 客户名称 |
| customer_phone | VARCHAR(30) | 否 | 客户联系电话 |
| planned_departure_time | DATETIME | 是 | 预计出发时间 |
| planned_arrival_time | DATETIME | 是 | 预计到达时间 |
| actual_departure_time | DATETIME | 否 | 实际出发时间 |
| actual_arrival_time | DATETIME | 否 | 实际到达时间 |
| status | VARCHAR(20) | 是 | 运输状态 |
| remark | VARCHAR(255) | 否 | 备注 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

业务约束建议：

1. `destination_port` 不能与 `origin_port` 相同。
2. `planned_arrival_time` 应晚于 `planned_departure_time`。
3. `cargo_weight` 应大于 0。
4. 运输任务创建时，船舶不能处于维修或停用状态。
5. 运输任务创建时，货物重量不能超过船舶载重量。

## 4.6 财务结算表 settlement

用于保存运输任务对应的费用结算信息。

```sql
CREATE TABLE settlement (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '结算ID',
  settlement_no VARCHAR(40) NOT NULL COMMENT '结算编号',
  transport_order_id BIGINT NOT NULL COMMENT '运输任务ID',
  customer_name VARCHAR(100) NOT NULL COMMENT '客户名称',
  freight_amount DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '运费金额',
  additional_fee DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '附加费用',
  discount_amount DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '优惠金额',
  receivable_amount DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '应收金额',
  received_amount DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '实收金额',
  status VARCHAR(20) NOT NULL DEFAULT 'UNSETTLED' COMMENT '结算状态：UNSETTLED未结算，PARTIAL部分结算，SETTLED已结算',
  settled_at DATETIME DEFAULT NULL COMMENT '结算完成时间',
  remark VARCHAR(255) DEFAULT NULL COMMENT '备注',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_settlement_no (settlement_no),
  UNIQUE KEY uk_settlement_transport_order_id (transport_order_id),
  KEY idx_settlement_customer_name (customer_name),
  KEY idx_settlement_status (status),
  KEY idx_settlement_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='财务结算表';
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 结算主键 |
| settlement_no | VARCHAR(40) | 是 | 结算编号，系统内唯一 |
| transport_order_id | BIGINT | 是 | 运输任务 ID |
| customer_name | VARCHAR(100) | 是 | 客户名称 |
| freight_amount | DECIMAL(12,2) | 是 | 运费金额 |
| additional_fee | DECIMAL(12,2) | 是 | 附加费用 |
| discount_amount | DECIMAL(12,2) | 是 | 优惠金额 |
| receivable_amount | DECIMAL(12,2) | 是 | 应收金额 |
| received_amount | DECIMAL(12,2) | 是 | 实收金额 |
| status | VARCHAR(20) | 是 | 结算状态 |
| settled_at | DATETIME | 否 | 结算完成时间 |
| remark | VARCHAR(255) | 否 | 备注 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

金额计算规则：

```text
receivable_amount = freight_amount + additional_fee - discount_amount
```

状态计算规则：

1. `received_amount = 0` 时，状态为 `UNSETTLED`。
2. `0 < received_amount < receivable_amount` 时，状态为 `PARTIAL`。
3. `received_amount = receivable_amount` 时，状态为 `SETTLED`。

## 5. 可选外键约束

如果希望数据库层保证关联完整性，可增加以下外键。课程设计中也可以不添加外键，只在后端业务代码中校验。

```sql
ALTER TABLE sys_user
  ADD CONSTRAINT fk_sys_user_role
  FOREIGN KEY (role_id) REFERENCES sys_role(id);

ALTER TABLE crew_member
  ADD CONSTRAINT fk_crew_member_ship
  FOREIGN KEY (ship_id) REFERENCES ship(id);

ALTER TABLE transport_order
  ADD CONSTRAINT fk_transport_order_ship
  FOREIGN KEY (ship_id) REFERENCES ship(id);

ALTER TABLE settlement
  ADD CONSTRAINT fk_settlement_transport_order
  FOREIGN KEY (transport_order_id) REFERENCES transport_order(id);
```

外键使用建议：

1. 如果开发时频繁删除测试数据，可以暂不创建外键，避免调试成本过高。
2. 如果希望数据关系更严谨，可以创建外键，但删除数据时需要注意关联约束。
3. 课程设计答辩中，只要能说明表之间逻辑关系即可，不强制必须使用数据库外键。

## 8. 首页统计数据来源

首页统计接口可直接从现有业务表聚合获取，不需要单独设计统计表。

| 统计项 | 数据来源 |
| --- | --- |
| shipCount | `ship` 表总数 |
| crewCount | `crew_member` 表总数 |
| pendingTransportCount | `transport_order` 表中 `status = 'PENDING'` 的数量 |
| inTransitCount | `transport_order` 表中 `status = 'IN_TRANSIT'` 的数量 |
| unsettledCount | `settlement` 表中 `status != 'SETTLED'` 的数量 |
| totalReceivableAmount | `settlement.receivable_amount` 汇总 |
| totalReceivedAmount | `settlement.received_amount` 汇总 |

## 9. 设计取舍总结

本数据库设计保留了航运公司管理系统的核心业务数据关系，同时避免了课程设计中投入产出比较低的复杂设计。

本项目暂不设计：

1. 密码加密、哈希、盐值和复杂认证表。
2. 操作日志表。
3. 复杂审批流表。
4. 多租户、组织架构和数据权限表。
5. 航线轨迹、实时定位和地图坐标表。
6. 在线支付、发票和复杂财务凭证表。

后续开发时应优先保证核心表的增删改查、关联查询和前后端接口联通。
