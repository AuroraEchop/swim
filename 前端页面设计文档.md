# 简易航运公司管理系统前端页面设计文档

## 1. 文档说明

本文档用于指导 Vue 3 前端开发，页面、路由、字段和交互均以 [API接口文档.md](API接口文档.md) 与 [DTO设计文档.md](DTO设计文档.md) 为依据。前端目标是完成一个清晰、稳定、适合课程设计答辩展示的航运公司后台管理系统，而不是营销页或复杂商业系统。

旧版前端废案中的“演示环境”等弱化系统感的文案不再使用。界面应直接呈现为业务管理系统：用户登录后进入工作台，围绕船舶、船员、运输任务、财务结算和系统配置完成日常维护。

## 2. 设计目标

1. 业务闭环清晰：从船舶与船员维护，到运输任务创建、状态流转，再到财务结算。
2. 实现成本适中：优先使用 Element Plus 现成组件，不做高成本动效、地图、复杂权限或生产级安全能力。
3. 航运主题明确：通过命名、图标、状态标签、数据字段和少量海运色彩体现主题，不做夸张的大屏风格。
4. 管理效率优先：列表页统一使用筛选区、表格、分页、抽屉表单和确认弹窗，方便反复录入与修改。
5. 认知边界明确：登录和密码处理按课程设计简化方案实现，不在前端额外加密，不将明文密码方案标成漏洞。

## 3. 技术栈与目录规划

前端采用以下技术：

| 类型 | 方案 |
| --- | --- |
| 构建 | Vite |
| 框架 | Vue 3 + TypeScript |
| 路由 | Vue Router |
| 状态管理 | Pinia |
| 请求 | Axios |
| UI 组件 | Element Plus |
| 图标 | Element Plus Icons |
| 包管理 | pnpm |

建议目录结构：

```text
frontend/
├─ src/api/                 接口封装
├─ src/assets/              静态资源
├─ src/components/          通用业务组件
├─ src/layouts/             登录布局与后台布局
├─ src/router/              路由与守卫
├─ src/stores/              Pinia 状态
├─ src/styles/              全局样式与主题变量
├─ src/types/               DTO 类型定义
└─ src/views/               页面视图
```

## 4. 视觉与交互基调

界面采用偏办公系统的浅色风格，主色建议为深海青或蓝绿色，背景使用冷灰白，表格和表单保持高可读性。状态颜色保持常规认知：启用、空闲、已结算使用绿色；运输中、部分结算使用蓝色；待处理使用橙色；停用、取消、离职使用灰色或红色。

推荐文案：

| 位置 | 文案 |
| --- | --- |
| 系统名称 | 航运公司管理系统 |
| 登录页副标题 | 统一管理船舶、船员、运输任务与结算流程 |
| 首页标题 | 业务工作台 |
| 退出按钮 | 退出登录 |

需要避免的内容：

1. 不在界面中写“演示环境”“仅供展示”等降低完成度的文案。
2. 不做登陆前营销落地页，登录页就是系统入口。
3. 不做暗色大屏驾驶舱，课程设计后台应优先保证清楚、稳定、可操作。
4. 不把所有页面堆成卡片墙，业务列表页以表格和筛选为核心。

## 5. 全局应用结构

### 5.1 路由结构

| 路由 | 页面 | 说明 |
| --- | --- | --- |
| `/login` | 登录页 | 未登录访问系统时进入 |
| `/dashboard` | 业务工作台 | 登录后默认页 |
| `/ships` | 船舶管理 | 船舶档案维护 |
| `/crew-members` | 船员管理 | 船员档案与船舶分配 |
| `/transport-orders` | 运输任务 | 运输订单与状态流转 |
| `/settlements` | 财务结算 | 运费结算与收款维护 |
| `/dictionaries` | 基础字典 | 船舶类型、货物类型、港口、岗位 |
| `/users` | 用户管理 | 系统用户维护 |
| `/roles` | 角色管理 | 角色基础维护 |

### 5.2 后台布局

后台采用左侧菜单、顶部栏、主内容区的三段式布局。

左侧菜单分组：

| 分组 | 菜单 |
| --- | --- |
| 业务工作台 | 首页概览、船舶管理、船员管理、运输任务、财务结算 |
| 系统配置 | 基础字典、用户管理、角色管理 |

顶部栏展示当前页面标题、刷新按钮、登录用户姓名和用户下拉菜单。用户下拉菜单包含“修改密码”和“退出登录”。

### 5.3 登录状态

登录成功后保存 `loginToken` 与用户基础信息到 `localStorage`。Axios 请求拦截器统一添加：

```http
Authorization: Bearer <loginToken>
```

路由守卫逻辑：

1. 未登录访问后台页面，跳转 `/login`。
2. 已登录访问 `/login`，跳转 `/dashboard`。
3. 刷新页面后调用 `GET /auth/me` 恢复当前用户。
4. 接口返回 `401` 时清除本地登录信息并跳转 `/login`。

课程设计说明：登录 Token 使用后端返回的 `demo-token-{username}-{id}`，密码按明文提交和明文比对。前端不做哈希、加盐、JWT 刷新等生产级安全功能，这是课程设计范围内的有意简化。

## 6. 页面设计

### 6.1 登录页

路由：`/login`

接口：`POST /auth/login`

页面结构：

1. 左侧或顶部展示系统名称“航运公司管理系统”和一句业务副标题。
2. 登录表单包含用户名、密码、登录按钮。
3. 表单校验：用户名必填，密码必填且 6 至 30 位。
4. 登录失败使用 Element Plus message 展示后端错误信息。
5. 登录成功后保存 token 和用户信息，进入 `/dashboard`。

密码字段按普通密码输入框处理，提交值即明文密码。此设计与后端和数据库文档一致，是课程设计简化方案。

### 6.2 业务工作台

路由：`/dashboard`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /dashboard/summary` | 获取统计数据 |
| `GET /dashboard/recent-transport-orders?limit=5` | 获取运输任务动态 |

页面内容：

1. 顶部 KPI：船舶数量、船员数量、待出发任务、运输中任务、未结清记录。
2. 金额概览：应收总金额、实收总金额、回款比例。
3. 运输任务动态表：任务编号、货物、船舶、航线、状态、预计出发时间。
4. 快捷操作：新建运输任务、新增船舶、查看财务结算。

空数据状态显示“暂无业务数据”，并提供进入核心页面的操作按钮。

### 6.3 船舶管理

路由：`/ships`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /ships` | 分页查询 |
| `GET /ships/{id}` | 查询详情 |
| `POST /ships` | 新增船舶 |
| `PUT /ships/{id}` | 修改船舶 |
| `PATCH /ships/{id}/status` | 修改状态 |
| `DELETE /ships/{id}` | 删除船舶 |

筛选项：关键词、船舶类型、所属港口、船舶状态。

表格列：船舶编号、船名、船舶类型、载重量、所属港口、状态、创建时间、操作。

表单字段：船舶编号、船名、船舶类型、载重量、所属港口、状态、备注。编辑时不修改船舶编号。

操作：新增、编辑、状态调整、删除。删除前必须二次确认；对于已有业务关联的数据，优先引导改为停用。

### 6.4 船员管理

路由：`/crew-members`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /crew-members` | 分页查询 |
| `GET /crew-members/{id}` | 查询详情 |
| `POST /crew-members` | 新增船员 |
| `PUT /crew-members/{id}` | 修改船员 |
| `PATCH /crew-members/{id}/status` | 修改状态 |
| `DELETE /crew-members/{id}` | 删除船员 |
| `GET /ships?pageSize=100` | 船舶下拉选项 |

筛选项：关键词、岗位、所属船舶、船员状态。

表格列：船员编号、姓名、性别、联系电话、证件编号、岗位、所属船舶、状态、操作。

表单字段：船员编号、姓名、性别、联系电话、证件编号、岗位、所属船舶、状态、备注。编辑时不修改船员编号。

所属船舶使用远程下拉，展示船名，提交 `shipId`。未分配船舶时显示“未分配”。

### 6.5 运输任务

路由：`/transport-orders`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /transport-orders` | 分页查询 |
| `GET /transport-orders/{id}` | 查询详情 |
| `POST /transport-orders` | 新增任务 |
| `PUT /transport-orders/{id}` | 修改任务 |
| `PATCH /transport-orders/{id}/status` | 修改状态 |
| `DELETE /transport-orders/{id}` | 删除任务 |
| `GET /ships?pageSize=100` | 船舶下拉选项 |

筛选项：关键词、船舶、起运港、目的港、运输状态、预计出发日期范围。

表格列：任务编号、货物名称、货物类型、货物重量、航线、承运船舶、客户、预计出发、预计到达、运输状态、结算状态、操作。

表单字段：货物名称、货物类型、货物重量、起运港、目的港、承运船舶、客户名称、客户电话、预计出发时间、预计到达时间、备注。

状态流转：

| 当前状态 | 可操作 |
| --- | --- |
| `PENDING` | 开始运输、取消任务、编辑、删除 |
| `IN_TRANSIT` | 标记到达 |
| `ARRIVED` | 查看详情、创建或查看结算 |
| `CANCELLED` | 查看详情、删除 |

状态变为 `IN_TRANSIT` 时弹窗填写实际出发时间；状态变为 `ARRIVED` 时弹窗填写实际到达时间。任务详情使用抽屉展示，包含货物、客户、船舶、计划时间、实际时间和结算入口。

### 6.6 财务结算

路由：`/settlements`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /settlements` | 分页查询 |
| `GET /settlements/{id}` | 查询详情 |
| `POST /settlements` | 新增结算 |
| `PUT /settlements/{id}` | 修改结算 |
| `PATCH /settlements/{id}/payment` | 修改实收金额 |
| `DELETE /settlements/{id}` | 删除结算 |
| `GET /transport-orders` | 运输任务下拉选项 |

筛选项：关键词、客户名称、结算状态、创建日期范围。

表格列：结算编号、运输任务编号、客户、运费、附加费用、优惠金额、应收金额、实收金额、结算状态、结算时间、操作。

表单字段：运输任务、运费金额、附加费用、优惠金额、实收金额、备注。

金额规则在表单中实时展示：

```text
应收金额 = 运费金额 + 附加费用 - 优惠金额
```

收款操作单独使用小抽屉或弹窗，只填写实收金额和付款时间。后端根据实收金额返回结算状态，前端直接刷新当前行和统计数据。

| --- | --- |
| `GET /dictionaries/{type}` | 查询字典项 |

| `PUT /dictionaries/{id}` | 修改字典项 |

| 字典类型 | 页面名称 |
| --- | --- |
| `SHIP_TYPE` | 船舶类型 |
| `CARGO_TYPE` | 货物类型 |
| `PORT` | 港口 |
| `CREW_POSITION` | 船员岗位 |

表格列：显示名称、字典值、排序号、启用状态、备注、操作。

表单字段：显示名称、字典值、排序号、启用状态、备注。删除前提示该字典项可能已被业务数据引用，建议优先停用。

### 6.8 用户管理

路由：`/users`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /users` | 分页查询 |
| `GET /users/{id}` | 查询详情 |
| `POST /users` | 新增用户 |
| `PUT /users/{id}` | 修改用户 |
| `DELETE /users/{id}` | 删除用户 |
| `GET /roles` | 角色下拉选项 |

筛选项：用户名、真实姓名、角色、用户状态。

表格列：用户名、真实姓名、联系电话、邮箱、角色、状态、创建时间、操作。

新增字段：用户名、明文密码、真实姓名、联系电话、邮箱、角色、状态。

编辑字段：真实姓名、联系电话、邮箱、角色、状态。普通编辑不修改用户名和密码。

新增用户表单中可显示轻量说明：“本课程设计按简化方案保存普通文本密码”。这句话用于防止误判为未完成项，不要求引导用户做生产级安全配置。

### 6.9 角色管理

路由：`/roles`

接口：

| 接口 | 用途 |
| --- | --- |
| `GET /roles` | 查询角色 |
| `POST /roles` | 新增角色 |
| `PUT /roles/{id}` | 修改角色 |
| `DELETE /roles/{id}` | 删除角色 |

表格列：角色名称、角色编码、角色说明、内置角色、操作。

新增字段：角色名称、角色编码、角色说明。编辑时不修改角色编码。

`permissions` 字段可作为前端展示或预留字段，但后端不持久化权限明细。页面不设计复杂权限树，只根据角色编码完成菜单和按钮的基础控制。内置角色不允许删除。

### 6.10 修改密码

入口：顶部用户下拉菜单。

接口：`PUT /auth/password`

表单字段：旧密码、新密码、确认新密码。

校验规则：

1. 旧密码必填。
2. 新密码 6 至 30 位。
3. 确认新密码必须与新密码一致。
4. 新密码不能与旧密码相同。

提示文案采用课程设计口径：“密码按课程设计简化方案更新为普通文本”。不在界面中写安全警告。

## 8. 前端数据与接口封装

### 8.1 API 封装

按模块拆分：

```text
src/api/auth.ts
src/api/dashboard.ts
src/api/ships.ts
src/api/crewMembers.ts
src/api/transportOrders.ts
src/api/settlements.ts
src/api/dictionaries.ts
src/api/users.ts
src/api/roles.ts
```

Axios 统一处理：

1. `baseURL` 使用 `/api`，由 Vite 代理到 `http://localhost:8080`。
2. 请求前添加 `Authorization`。
3. 响应后剥离统一响应体中的 `data`。
4. `401` 自动退出登录。
5. 其他错误使用后端 `message` 展示。

### 8.2 状态管理

Pinia store 建议：

| Store | 内容 |
| --- | --- |
| `auth` | token、当前用户、登录、退出、刷新用户 |
| `dictionary` | 字典项缓存与刷新 |
| `app` | 菜单折叠、页面加载等轻量状态 |

业务列表数据保留在页面内，不放入全局 store。

## 9. 表单校验与状态展示

统一交互规则：

1. 列表加载使用 Element Plus 表格 loading。
2. 空列表展示空状态和主要操作按钮。
3. 新增、编辑、状态修改成功后刷新列表。
4. 删除、取消任务、收款等关键操作必须确认。
5. 金额、重量展示保留两位小数。
6. 日期时间提交格式为 `yyyy-MM-dd HH:mm:ss`。
7. 分页参数统一为 `page` 和 `pageSize`。

状态标签映射：

| 枚举 | 中文 |
| --- | --- |
| `IDLE` | 空闲 |
| `SAILING` | 运输中 |
| `MAINTENANCE` | 维修中 |
| `DISABLED` | 停用 |
| `ON_DUTY` | 在岗 |
| `ON_LEAVE` | 休假 |
| `UNASSIGNED` | 待分配 |
| `RESIGNED` | 离职 |
| `PENDING` | 待出发 |
| `IN_TRANSIT` | 运输中 |
| `ARRIVED` | 已到达 |
| `CANCELLED` | 已取消 |
| `UNSETTLED` | 未结算 |
| `PARTIAL` | 部分结算 |
| `SETTLED` | 已结算 |
| `ENABLED` | 启用 |
| `DISABLED` | 禁用 |

## 10. 响应式适配

系统以桌面端为主，兼顾窄屏可用性。

1. 宽度大于 1024px：左侧菜单固定，内容区展示完整表格。
2. 宽度 768px 至 1024px：菜单允许折叠，表格横向滚动。
3. 宽度小于 768px：表单抽屉宽度使用 100%，表格保留横向滚动，不强行改成卡片列表。

## 11. 开发顺序

1. 创建 Vite + Vue 3 + TypeScript 项目。
2. 安装 Pinia、Vue Router、Axios、Element Plus 和图标库。
3. 完成全局样式、Axios 封装、DTO 类型、路由守卫。
4. 完成登录页、后台布局、修改密码弹窗。
5. 完成业务工作台。
6. 完成字典接口和通用下拉组件。
7. 完成船舶管理。
8. 完成船员管理。
9. 完成运输任务。
10. 完成财务结算。
11. 完成用户管理和角色管理。
12. 统一检查空状态、错误提示、表单校验和响应式表现。

## 12. 验收清单

1. 未登录访问后台页面会跳转登录页。
2. 登录成功后可进入业务工作台并看到统计数据。
3. 所有列表页支持筛选、分页、刷新和空状态。
4. 船舶、船员、运输任务、结算、字典、用户、角色可完成对应增删改查。
